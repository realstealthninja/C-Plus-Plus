\doxysubsection{Detailed Description}
Counting Inversions using \href{https://en.wikipedia.org/wiki/Merge_sort}{\texttt{ Merge Sort}} 

Program to count the number of inversions in an array using merge-\/sort.

The count of inversions help to determine how close the array is to be sorted in ASCENDING order.

two elements a\mbox{[}i\mbox{]} and a\mbox{[}j\mbox{]} form an inversion if {\ttfamily a\mbox{[}i\mbox{]}} \texorpdfstring{$>$}{>} {\ttfamily a\mbox{[}j\mbox{]}} and i \texorpdfstring{$<$}{<} j

Time Complexity -\/-\/\texorpdfstring{$>$}{>} {\ttfamily O(n.\+log n)}

Space Complexity -\/-\/\texorpdfstring{$>$}{>} {\ttfamily O(n)} ; additional array {\ttfamily temp\mbox{[}1..n\mbox{]}} \hypertarget{count__inversions_8cpp_autotoc_md110}{}\doxysubsubsubsection{\texorpdfstring{Algorithm}{Algorithm}}\label{count__inversions_8cpp_autotoc_md110}

\begin{DoxyEnumerate}
\item The idea is similar to merge sort, divide the array into two equal or almost equal halves in each step until the base case is reached.
\item Create a function merge that counts the number of inversions when two halves of the array are merged, create two indices i and j, i is the index for first half and j is an index of the second half. if {\ttfamily a\mbox{[}i\mbox{]}} is greater than {\ttfamily a\mbox{[}j\mbox{]}}, then there are (mid – i) inversions, Because left and right subarrays are sorted, so all the remaining elements in left-\/subarray (a\mbox{[}i+1\mbox{]}, a\mbox{[}i+2\mbox{]} … a\mbox{[}mid\mbox{]}) will be greater than a\mbox{[}j\mbox{]}.
\item Create a recursive function to divide the array into halves and find the answer by summing the number of inversions is the first half, number of inversion in the second half and the number of inversions by merging the two.
\item The base case of recursion is when there is only one element in the given half.
\item Print the answer
\end{DoxyEnumerate}
