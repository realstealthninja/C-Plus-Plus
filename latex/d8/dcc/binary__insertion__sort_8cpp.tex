\doxysubsection{Detailed Description}
\href{https://en.wikipedia.org/wiki/Insertion_sort}{\texttt{ Binary Insertion Sort Algorithm (Insertion Sort)}} 

If the cost of comparisons exceeds the cost of swaps, as is the case for example with string keys stored by reference or with human interaction (such as choosing one of a pair displayed side-\/by-\/side), then using binary insertion sort may yield better performance. Binary insertion sort employs a binary search to determine the correct location to insert new elements, and therefore performs ⌈log2 n⌉ comparisons in the worst case. When each element in the array is searched for and inserted this is O(n log n). The algorithm as a whole still has a running time of O(n2) on average because of the series
\begin{DoxyItemize}
\item of swaps required for each insertion. However it has several advantages such as
\end{DoxyItemize}
\begin{DoxyEnumerate}
\item Easy to implement
\item For small set of data it is quite efficient
\item More efficient that other Quadratic complexity algorithms like Selection sort or bubble sort.
\item It is efficient to use it when the cost of comparison is high.
\item It\textquotesingle{}s stable that is it does not change the relative order of elements with equal keys.
\item It can sort the array or list as it receives.
\end{DoxyEnumerate}

Example execution steps\+:
\begin{DoxyEnumerate}
\item Suppose initially we have \begin{bmatrix}40 &30 &20 &50 &10\end{bmatrix}
\item We start traversing from 40 till we reach 10 when we reach at 30 we find that it is not at it\textquotesingle{}s correct place so we take 30 and place it at a correct position thus the array will become \begin{bmatrix}30 &40 &20 &50 &10\end{bmatrix}
\item In the next iteration we are at 20 we find that this is also misplaced so we place it at the correct sorted position thus the array in this iteration becomes \begin{bmatrix}20 &30 &40 &50 &10\end{bmatrix}
\item We do not do anything with 50 and move on to the next iteration and select 10 which is misplaced and place it at correct position. Thus, we have \begin{bmatrix}10 &20 &30 &40 &50\end{bmatrix} 
\end{DoxyEnumerate}
